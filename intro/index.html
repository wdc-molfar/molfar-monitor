# Програмний модуль "molfar-monitor" | Вступ

Програмний модуль molfar-monitor (opens new window) – "Програмний модуль для моніторингу стану мікросервісів" - це RESTfull сервіс, написаний мовою програмування JavaScript, який надає можливості для моніторингу стану мікросервісів, зокрема: обробляти запити від мікросервісів для додавання їх до реєстру моніторингу, надсилання запитів до мікросервісів з метою визначити їх стан, доступність та працездатність, видаляти мікросервіси з реєстру моніторингу; та надає можливості управління мікросервісами, зокрема: перенаправляти запити до molfar-node (opens new window) на розгортання, запуск, налаштування, зупинку та згортання мікросервісів; а також зберігає інформацію про поточний стан всіх зареєстрованих мікросервісів.

Функціональне призначення

Програмний модуль призначений для моніторингу стану мікросервісів, зокрема надає можливості для обробки запитів від мікросервісів для додавання їх до реєстру моніторингу, надсилання запитів до мікросервісів з метою визначити їх стан, доступність та працездатність, видаляти мікросервіси з реєстру моніторингу; а також призначений для управління мікросервісами, і зокрема дозволяє перенаправляти запити до molfar-node (opens new window) на розгортання, запуск, налаштування, зупинку та згортання мікросервісів. Також програмний модуль зберігає інформацію про поточний стан всіх зареєстрованих мікросервісів.

Опис логічної структури

Програмний модуль складається з:

  • javascript – програмний підмодуль, що приймає та оброблює запити

Використовувані технічні засоби

Програмний модуль експлуатується на сервері під управлінням Node.js, а також HTTP-клієнт axios на основі промісів для нього. В основі управління викликів з теміналу є менеджер пакетів npm. В основі управління всіх сервісів є система оркестрації Kubernetes, де всі контейнери працюють з використанням Docker.

Виклик та завантаження

Для запуску програмного модуля molfar-monitor необхідно:

  1. Клонувати репозиторій:
git clone https://github.com/wdc-molfar/molfar-monitor.git
  1. Перейти в директорію склонованого репозиторія та проінсталювати всі залежності:
cd /molfar-monitor
npm install

Завантаження програмного модуля забезпечується введенням в WEB-браузері адреси завантажувального модуля http://{hostname} (opens new window) з можливими вказівками:

  • /state (opens new window) - метод GET для отримання інформації про стан та працездатність контейнера
  • /node/register (opens new window) - метод POST для додавання вузла в реєстр моніторингу
  • /node/unregister (opens new window) - метод POST для видалення вузла з реєстру моніторингу
  • /node/state/:instance (opens new window) - метод GET для отримання інформації про стан та працездатність окремого вузла за ідентифікатором instance
  • /node/deploy/:instance (opens new window) - метод POST для направлення запиту на завантаження сервісу в контейнер за ідентифікатором вузла instance (розгортання сервісу) та повернення інформації про результат завантаження
  • /ms/start/:id (opens new window) - метод POST для направлення запиту на запуск сервісу за ідентифікатором id та повернення інформації про запуск
  • /ms/config/:id (opens new window) - метод POST для направлення запиту на конфігурацію сервісу за ідентифікатором id та повернення інформації про конфігурацію
  • /ms/metrics/:instance (opens new window) - метод GET для отримання метрик сервісу за ідентифікатором id
  • /ms/terminate/:id (opens new window) - метод POST для направлення запиту на запинку сервісу за ідентифікатором id та повернення інформації про зупинку
  • /node/undeploy/:instance (opens new window) - метод POST для направлення запиту на вивантаження сервісу з контейнера за ідентифікатором вузла instance (згортання сервісу) та повернення інформації про результат вивантаження
